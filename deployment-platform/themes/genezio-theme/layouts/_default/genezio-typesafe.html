FAQ

Your questions answered

{{/* Question 1 */}}

DeployApps is used by our clients to build and host full-stack apps. These include a variety of full-stack Web Apps, Mobile Apps, Enterprise automation apps, and Blockchain enabled apps. Also, some developers use DeployApps just to set up scheduled tasks or to keep an API key private in a part of their app that runs in the background. To learn more and see examples, check out our docs.
{{/* Question 2 */}}

Your DevOps team is probably pretty busy, so in order to try out a pilot project, it's much easier to use our serverless approach.
{{/* Question 3 */}}

tRPC is an amazing library to add type safe calls to TypeScript frontends and backends. DeployApps.RPC allows developers to write the backend in, say, Go, and call the exported type safe API from a react frontend.
{{/* Question 4 */}}

REST is a flexible standard that is very well suited to open APIs. We have created DeployApps.RPC to allow developers to design a backend that doesn't necessarily fit into the REST paradigm. We see more and more developers interested in ensuring type safety and consistent design for the frontend-backend communication. This is the best way to ensure less things break as they change things.
{{/* Question 5 */}}

DeployApps.Services is similar to a BaaS in the sense that offers a wide variety of out-of-the-box services to help you be more productive. Implementing authentication, setting up cron jobs and working with databases and queues are one click away. They are designed to save you time and effort when developing and maintaining your applications. The differences steam from the fact that DeployApps enables you to write your own backend business logic. Also, in DeployApps the database is not exposed directly to the frontend, enhancing security and separation of concern.
{{/* Question 6 */}}

DeployApps.Cloud ensures that you can deploy applications in a serverless fashion, and instantly have a secure, publicly available deployment. After you create, build, and test your application locally, you can deploy it to DeployApps Cloud. You can now deploy on production on our managed AWS account. You can also start experimenting on our own serverless cloud that offers zero cold start penalty for backend functions and lower, predictable cost.
{{/* Question 7 */}}

You can build your apps using TypeScript for both backend and frontend, which is tested and production-ready. We also offer beta support for Kotlin and DART, and are working towards enabling support for GoLang and Swift.
{{/* Question 8 */}}

DeployApps is compatible with various frameworks like React, Vue, Angular, Svelte, Flutter, ExpressJS, Fastify, and more, thanks to its framework-agnostic nature.
{{/* Question 9 */}}

If all you need is a database exposed to the frontend, then DeployApps is not very useful. However, when your business logic becomes more complex, you will need to implement it in a backend, and DeployApps is a perfect solution for that.
{{/* Question 11 */}}

Yes, you can try out DeployApps locally without an account. An account is only required if you wish to deploy a project on our platform. See our docs for how to do this.
{{/* Question 12 */}}

If your application requires more than a database, DeployApps is the ideal solution. Here are some key benefits:
  1. Flexibility: DeployApps supports various programming languages, making it more versatile for developers with different tech stacks.
  2. Type Safety: DeployApps provides a type safe interface, reducing errors and enhancing code quality.
  3. Full-Stack Support: You can connect DeployApps with any frontend, whether it's for web, mobile, or desktop applications, and still maintain type safety.
  4. Productivity: DeployApps offers a range of built-in services like authentication, cron jobs and database integration, making development faster and more efficient.
  5. Scalability: DeployApps ensures auto-scalability for your applications, adapting to traffic demands effortlessly.
